1️⃣ CIFRADO DE DISCO COMPLETO (LUKS)

Comando Función Cómo se Aplica Comandos Previos / Requisitos
sudo lsblk
 Listar dispositivos de almacenamiento disponibles Ejecutar antes de encriptar para identificar la partición correcta (ej: /dev/sda2)
✓ Usuario con permisos sudo
✓ Acceso físico o administrativo al sistema
sudo cryptsetup luksFormat /dev/sda2
 Formatea una partición con cifrado LUKS y establece una contraseña maestra En TechCorp: Aplicado a laptops de auditores para proteger datos en reposo
Paso: El sistema solicita confirmación (escribir "yes"), luego pide una contraseña fuerte (mín. 8 caracteres)
✓ sudo lsblk (para identificar /dev/sda2)
⚠️ ADVERTENCIA: Borra todos los datos de la partición
⚠️ Hacer backup antes de ejecutar
sudo cryptsetup luksOpen /dev/sda2 disco_seguro
 Desbloquea una partición LUKS cifrada y la monta como "disco_seguro" Ejecutado al arrancar el sistema o cuando se necesita acceder a los datos
Flujo: El sistema solicita la contraseña maestra → Se desbloquea → Aparece en /dev/mapper/disco_seguro
✓ Partición previamente formateada con luksFormat
✓ Conocer la contraseña LUKS
sudo mkfs.ext4 /dev/mapper/disco_seguro
 Crea un sistema de archivos ext4 dentro de la partición LUKS desbloquada Después de desbloquear LUKS, se necesita crear un sistema de archivos para poder guardar datos
Secuencia: luksOpen → mkfs.ext4 → mount → usar
✓ luksOpen ya ejecutado
✓ /dev/mapper/disco_seguro debe existir
sudo mount /dev/mapper/disco_seguro /mnt/datos_seguros
 Monta la partición cifrada en un punto de acceso del sistema de archivos Permite acceder a los datos cifrados desde el directorio /mnt/datos_seguros
En TechCorp: Los auditores acceden a reportes financieros desde /mnt/datos_seguros
✓ mkfs.ext4 ya ejecutado
✓ Crear directorio: sudo mkdir -p /mnt/datos_seguros
sudo umount /mnt/datos_seguros
 Desmonta la partición cifrada (necesario antes de cerrar LUKS) Ejecutar cuando se finaliza el trabajo con datos cifrados
Libera recursos y prepara para desconectar luksClose
✓ No debe haber archivos abiertos de ese directorio
sudo cryptsetup luksClose disco_seguro
 Cierra y bloquea la partición LUKS (vuelve a cifrarla) Cierra el acceso a los datos, volviéndolos inaccesibles hasta el próximo luksOpen
Se debe ejecutar después de umount
✓ umount /dev/mapper/disco_seguro
Flujo Completo para TechCorp:
1. sudo lsblk
2. sudo cryptsetup luksFormat /dev/sda2
3. sudo cryptsetup luksOpen /dev/sda2 disco_seguro
4. sudo mkfs.ext4 /dev/mapper/disco_seguro
5. sudo mount /dev/mapper/disco_seguro /mnt/datos_seguros
6. [Trabajar con datos]
7. sudo umount /mnt/datos_seguros
8. sudo cryptsetup luksClose disco_seguro

2️⃣ CONTROL DE ACCESOS (ACL - Access Control Lists)

Comando Función Cómo se Aplica Comandos Previos / Requisitos
getfacl /ruta/archivo
 Visualiza los permisos ACL actuales de un archivo o directorio Verificar qué usuarios tienen acceso a un recurso
En TechCorp: getfacl /srv/datos/reportes
✓ Archivo/directorio debe existir
✓ Tener permiso de lectura en el elemento padre
setfacl -m u:usuario:rwx /ruta
 Asigna permisos ACL específicos a un usuario individual (r=read, w=write, x=execute) Permite control granular de accesos sin modificar permisos tradicionales
Caso TechCorp:
setfacl -m u:gerente:rwx /srv/datos/reportes
(Solo gerente tiene rwx, otros usuarios no)
✓ Usuario debe existir: id usuario
✓ Tener permisos sudo
setfacl -m g:grupo:rx /ruta
 Asigna permisos ACL a un grupo específico Aplicar permisos a múltiples usuarios agrupados
Ejemplo: setfacl -m g:auditores:rx /srv/datos/reportes
(Todos en grupo 'auditores' pueden leer y ejecutar)
✓ Grupo debe existir: getent group grupo
✓ Usuarios deben estar en el grupo
setfacl -m u:usuario:- /ruta
 Revoca todos los permisos de un usuario Denegar acceso completo a un usuario específico
El símbolo "-" significa sin permisos
✓ Usuario debe estar en ACL actual
setfacl -x u:usuario /ruta
 Elimina completamente la entrada ACL de un usuario Remover usuario del control de acceso (vuelve a permisos por defecto)
Diferencia: "-" = sin permisos, "-x" = remove entry
✓ Usuario debe estar en la ACL
setfacl -R -m u:usuario:rwx /ruta
 Aplica permisos ACL de forma RECURSIVA a un directorio y sus subdirectorios Aplicar permisos en toda una estructura de carpetas
Caso real:
setfacl -R -m u:gerente:rwx /srv/datos/
(Aplica permisos a todas las carpetas dentro)
✓ Parámetro -R (recursivo)
✓ Tener permisos en directorio padre
chmod 755 /ruta && setfacl -m u:usuario:rwx /ruta
 Combina permisos tradicionales (chmod) con ACL granulares Establecer una base de permisos y luego refinar con ACL
Primero chmod define base, luego setfacl agrega excepciones
✓ Ser propietario o root
Secuencia Típica para Gestión de Reportes en TechCorp:
1. sudo mkdir -p /srv/datos/reportes
2. sudo chmod 700 /srv/datos/reportes (permiso base)
3. getfacl /srv/datos/reportes (verificar estado)
4. sudo setfacl -m u:gerente:rwx /srv/datos/reportes
5. sudo setfacl -m g:auditores:rx /srv/datos/reportes
6. getfacl /srv/datos/reportes (verificar cambios)

3️⃣ CIFRADO Y FIRMA DIGITAL CON GPG

Comando Función Cómo se Aplica Comandos Previos / Requisitos
gpg --version
 Verifica si GPG está instalado y muestra la versión Comando de diagnóstico previo
✓ Sistema Linux/Ubuntu
Si no aparece: sudo apt-get install gnupg
sudo apt-get install gnupg
 Instala la herramienta GPG en Ubuntu (si no está) Instalación inicial en un sistema sin GPG
✓ Acceso a internet
✓ Permisos sudo
gpg --gen-key
 Genera un nuevo par de claves (pública/privada) para el usuario Crear identidad de usuario en GPG
Proceso interactivo:
• Tipo de clave (RSA)
• Tamaño (2048 o 4096 bits)
• Nombre, email, comentario
• Contraseña de la clave privada
✓ GPG instalado
Se ejecuta una sola vez por usuario
gpg -c archivo.pdf
 Cifra un archivo de forma SIMÉTRICA con una contraseña (sin generar claves) En TechCorp: Auditores cifran reportes rápidamente
Uso: gpg -c reporte_financiero.pdf
• Solicita contraseña
• Genera reporte_financiero.pdf.gpg
✓ Archivo debe existir
✓ GPG instalado
No requiere generar claves
gpg -d archivo.pdf.gpg
 Descifra un archivo cifrado simétricamente Recuperar contenido de archivos cifrados
Flujo:
gpg -d reporte_financiero.pdf.gpg
[Pide contraseña]
[Muestra contenido desencriptado]
✓ Conocer la contraseña
✓ Archivo .gpg debe existir
gpg -d archivo.pdf.gpg > archivo_descifrado.pdf
 Descifra y guarda el contenido en un nuevo archivo Crear archivo legible a partir del cifrado
El ">" redirige la salida a un archivo
✓ Descifrado previo funcionando
gpg -r usuario -e archivo.pdf
 Cifra un archivo ASIMÉTRICAMENTE para un usuario específico Enviar archivos seguros a otros usuarios
Caso: gpg -r gerente -e reporte.pdf
(Solo la clave pública de 'gerente' puede descifrar)
✓ Clave pública del usuario debe estar importada
✓ gpg --gen-key ejecutado previamente
gpg -s archivo.pdf
 Firma digitalmente un archivo (prueba autenticidad y no-repudio) Garantizar que un documento proviene de una fuente confiable
Crea archivo.pdf.gpg con firma digital
✓ gpg --gen-key ejecutado
✓ Contraseña de clave privada
gpg --verify archivo.pdf.gpg
 Verifica la firma digital de un archivo Confirmar que un documento no ha sido alterado
Muestra: Good signature / Bad signature
✓ Clave pública del firmante debe estar en el sistema
gpg --list-keys
 Lista todas las claves públicas disponibles en el sistema Verificar usuarios con los que se puede cifrar
Muestra: email, fingerprint, fecha de creación
✓ gpg --gen-key ejecutado previamente
gpg --list-secret-keys
 Lista todas las claves PRIVADAS del usuario actual Identificar claves disponibles para descifrar/firmar
Información sensible - no compartir
✓ Ser el propietario de las claves
gpg --export -a usuario > clave_publica.asc
 Exporta la clave pública de un usuario en formato ASCII Compartir clave pública con otros para que cifren archivos para ti
Distribución:
gpg --export -a gerente > gerente_clave_publica.asc
[Enviar archivo .asc a otros usuarios]
✓ Usuario debe existir en gpg --list-keys
gpg --import clave_publica.asc
 Importa una clave pública de otro usuario Agregar claves de otros usuarios para cifrar archivos para ellos
Después: gpg --list-keys mostrará la nueva clave
✓ Archivo .asc debe existir
✓ Tener archivo de clave pública válida
Flujo Completo para Cifrado en TechCorp:

Paso 1: Preparación (primera vez)
1. gpg --version
2. gpg --gen-key (cada usuario)

Paso 2: Compartir Claves
3. gpg --list-keys
4. gpg --export -a gerente > gerente.asc
5. [Enviar gerente.asc a auditores]
6. gpg --import gerente.asc

Paso 3: Cifrar Reportes
7. gpg -c reporte_financiero.pdf
8. [Introduce contraseña]
9. [Genera reporte_financiero.pdf.gpg]

Paso 4: Descifrar Reportes
10. gpg -d reporte_financiero.pdf.gpg
11. [Introduce contraseña]
12. [Accedes al contenido]

4️⃣ FUNCIONES HASH Y VERIFICACIÓN DE INTEGRIDAD

Comando Función Cómo se Aplica Comandos Previos / Requisitos
md5sum archivo
 Calcula el hash MD5 de un archivo (DEPRECATED - no usar en producción) Verificación básica de integridad (principalmente educativo ahora)
⚠️ MD5 es débil - usar SHA256 en su lugar
sha256sum archivo
 Calcula el hash SHA-256 de un archivo (estándar moderno) Verificar integridad de archivos importantes
Uso en TechCorp:
sha256sum reporte_financiero.pdf
[Output: abc123...xyz reporte_financiero.pdf]
✓ Archivo debe existir
Disponible por defecto en Ubuntu
sha256sum archivo > archivo.sha256
 Guarda el hash SHA-256 en un archivo de verificación Crear registro de integridad para auditoría
Se puede comparar después con sha256sum -c
✓ Archivo original debe existir
sha256sum -c archivo.sha256
 Verifica que el archivo actual coincida con su hash almacenado Detectar si un archivo ha sido modificado
Output:
reporte_financiero.pdf: OK ✓
o
reporte_financiero.pdf: FAILED ✗
✓ Archivo .sha256 debe existir
✓ Archivo original en el mismo directorio
sha512sum archivo
 Calcula el hash SHA-512 (más seguro que SHA-256) Máximo nivel de seguridad en verificación
Recomendado para datos ultra-sensibles
✓ Archivo debe existir
openssl dgst -sha256 archivo
 Alternativa: calcula hash SHA-256 usando OpenSSL Herramienta más versátil que sha256sum
✓ OpenSSL instalado
Flujo de Integridad en TechCorp:
1. sha256sum reporte_financiero.pdf > reporte.sha256
2. [Guardar reporte.sha256 en lugar seguro]
3. [Después de transferir el archivo...]
4. sha256sum -c reporte.sha256
5. [Verificar: OK significa que no ha sido alterado]

5️⃣ AUTENTICACIÓN Y GESTIÓN DE CONTRASEÑAS

Comando Función Cómo se Aplica Comandos Previos / Requisitos
sudo useradd -m usuario_nuevo
 Crea un nuevo usuario en el sistema Crear usuarios para auditores, gerentes, etc.
En TechCorp:
sudo useradd -m gerente
sudo useradd -m auditor1
✓ Permisos sudo
sudo passwd usuario
 Establece o cambia la contraseña de un usuario Definir contraseña segura con hash automático
El sistema aplica automáticamente salt y hash
✓ Usuario debe existir
✓ Permisos sudo
sudo usermod -aG grupo usuario
 Agrega un usuario a un grupo (para ACL y permisos) Asignar usuarios a grupos para gestión de accesos
Ejemplo:
sudo usermod -aG auditores auditor1
(auditor1 ahora hereda permisos del grupo 'auditores')
✓ Usuario debe existir
✓ Grupo debe existir: sudo groupadd auditores
sudo groupadd grupo_nuevo
 Crea un nuevo grupo Agrupar usuarios para simplificar permisos
En TechCorp:
sudo groupadd auditores
sudo groupadd gerentes
✓ Permisos sudo
id usuario
 Muestra UID, GID y grupos de un usuario Verificar membresía en grupos
Útil para confirmar que setfacl funcionó correctamente
✓ Usuario debe existir
sudo cat /etc/shadow
 Muestra archivo con contraseñas hasheadas (lectura solo root) Auditoría: verificar cómo se almacenan contraseñas
⚠️ Datos ALTAMENTE sensibles - solo administradores
✓ Permisos root (sudo)
Desde la carpeta /root
Flujo de Configuración de Usuarios en TechCorp:
1. sudo groupadd auditores
2. sudo groupadd gerentes
3. sudo useradd -m auditor1
4. sudo useradd -m gerente
5. sudo passwd auditor1
6. sudo passwd gerente
7. sudo usermod -aG auditores auditor1
8. sudo usermod -aG gerentes gerente
9. id auditor1 (verificar membresía)
10. [Luego aplicar ACL a directorios con setfacl]

6️⃣ HERRAMIENTAS CRIPTOGRÁFICAS: OpenSSL

Comando Función Cómo se Aplica Comandos Previos / Requisitos
openssl version
 Muestra versión de OpenSSL instalada Diagnóstico previo de disponibilidad
✓ OpenSSL instalado
Típicamente pre-instalado en Ubuntu
openssl enc -aes-256-cbc -salt -in archiv.txt -out archivo.enc
 Cifra un archivo con AES-256 en modo CBC (criptografía simétrica) Cifrar archivos con algoritmo fuerte
Variante interactiva:
openssl enc -aes-256-cbc -e -in reporte.pdf -out reporte.pdf.enc
[Pide contraseña]
[Crea reporte.pdf.enc]
✓ OpenSSL instalado
✓ Archivo de entrada debe existir
openssl enc -aes-256-cbc -d -in archivo.enc -out archivo
 Descifra un archivo previamente cifrado con AES-256 Recuperar contenido de archivos cifrados
El parámetro -d = decrypt
✓ Archivo .enc debe existir
✓ Conocer la contraseña
openssl req -x509 -newkey rsa:4096 -keyout clave.key -out cert.crt -days 365
 Genera un certificado X.509 auto-firmado y su clave privada Crear certificados para HTTPS/TLS en servidores
Válido por 365 días
✓ OpenSSL instalado
openssl s_client -connect servidor:443
 Verifica conexión SSL/TLS a un servidor remoto Validar que HTTPS funciona correctamente
Muestra cadena de certificados y detalles TLS
✓ Conexión a internet
✓ Servidor debe estar activo

⚡ GUÍA RÁPIDA: FLUJO COMPLETO PARA TECHCORP

Día 1: Configuración Inicial

1. Crear usuarios y grupos:
sudo groupadd auditores
sudo groupadd gerentes
sudo useradd -m auditor1 auditor2
sudo useradd -m gerente
sudo passwd auditor1 auditor2 gerente
2. Preparar estructura de almacenamiento:
sudo mkdir -p /srv/datos/reportes
sudo chmod 700 /srv/datos/reportes
3. Aplicar ACL para control de acceso:
sudo setfacl -m u:gerente:rwx /srv/datos/reportes
sudo setfacl -m g:auditores:rx /srv/datos/reportes
getfacl /srv/datos/reportes
4. Preparar cifrado de disco:
sudo lsblk
sudo cryptsetup luksFormat /dev/sda2
sudo cryptsetup luksOpen /dev/sda2 disco_seguro
sudo mkfs.ext4 /dev/mapper/disco_seguro
sudo mount /dev/mapper/disco_seguro /mnt/datos_seguros

Día 2: Uso Diario

5. Generar claves GPG (cada usuario):
gpg --gen-key
# [Responder preguntas interactivas]
gpg --list-keys
6. Cifrar un reporte:
gpg -c reporte_financiero.pdf
# [Introduce contraseña]
# Resultado: reporte_financiero.pdf.gpg
7. Guardar hash para verificación:
sha256sum reporte_financiero.pdf > reporte.sha256
8. Descifrar cuando se recibe:
gpg -d reporte_financiero.pdf.gpg > reporte_descifrado.pdf
sha256sum -c reporte.sha256

Operación Continua

9. Montar y desmontar disco cifrado:
# Inicio del día:
sudo cryptsetup luksOpen /dev/sda2 disco_seguro
sudo mount /dev/mapper/disco_seguro /mnt/datos_seguros

# Fin del día:
sudo umount /mnt/datos_seguros
sudo cryptsetup luksClose disco_seguro
